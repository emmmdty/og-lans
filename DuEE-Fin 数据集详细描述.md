## 1. 数据集概览

**DuEE-Fin** (Document-level Event Extraction in Financial domain) 是由百度发布的一个大规模、人工标注的中文篇章级金融事件抽取数据集。该数据集旨在解决文档级事件抽取 (DEE) 任务中数据稀缺的问题，并针对真实金融场景中的多事件、多值论元等复杂性提出了新的挑战。

* **领域**：真实的中文金融新闻和公告
* **规模**：
    * **文档总数**：11,699 篇非重复文档
    * **事件总数**：15,850 个事件
    * **论元总数**：81,632 个事件论元
* **Schema 规模**：13 种事件类型，92 种论元角色
* **主要挑战**：
    * **多事件识别 (MER)**：29.2% 的文档包含多个事件。
    * **多值论元识别 (MAER)**：16.8% 的事件包含多值论元（即同一角色对应多个实体）。
    * **长文本建模**：文档平均长度 499 字符，最长超 3000 字符，论元可能跨句分布。

## 2. 数据集划分与统计

数据集划分为训练集 (Train)、验证集 (Dev) 和测试集 (Test)。以下是详细统计信息：

| 数据集 | 文档数 (Documents) | 事件数 (Events) | 论元数 (Arguments) | 多事件文档比例 (MER) | 多值论元事件比例 (MAER) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Train** | 7,015 | 7,015 | 48,891 | 29.1% | 16.7% |
| **Dev** | 1,171 | 1,171 | 7,915 | 27.8% | 16.2% |
| **Test** | 3,513 | 3,513 | 24,826 | 30.0% | 17.2% |
| **Total** | **11,699** | **11,699** | **81,632** | **29.2%** | **16.8%** |

*注：在一般描述中提到每个文档平均包含 1.4 个事件。*

## 3. Schema 定义 (事件类型与角色)

DuEE-Fin 包含 **13** 种事件类型和 **92** 种论元角色。以下是所有事件类型及其对应的角色定义：

### 事件类型分布
1.  **股份回购 (Equity Repurchase)** - 13.9%
2.  **企业亏损 (Enterprise Loss)** - 11.1%
3.  **质押 (Equity Pledge)** - 10.7%
4.  **企业收购 (Enterprise Acquisition)** - 10.1%
5.  **高管变动 (Senior Manager Changes)** - 9.3%
6.  **中标 (Bidding)** - 9.3%
7.  **解除质押 (Cancellation of Pledge)** - 7.5%
8.  **股东减持 (Stock Reduction)** - 9.3%
9.  **企业融资 (Financing)** - 5.6%
10. **公司上市 (Company Listing)** - 5.3%
11. **股东增持 (Equity Overweight)** - 3.5%
12. **企业破产 (Go Bankruptcy)** - 2.4%
13. **被约谈 (Be Grilled)** - 1.6%

### 完整 Schema 描述

* **质押 (Equity Pledge)**
    * **角色**：质押方, 质权方, 质押物, 质押股票/股份数量, 质押物占持股比, 质押物占总股比, 质押物所属公司, 披露时间, 事件时间

* **股份回购 (Equity Repurchase)**
    * **角色**：回购方, 回购金额, 回购股份数量, 占公司总股本比例, 回购完成时间, 披露时间, 事件时间

* **企业融资 (Financing)**
    * **角色**：被投资方, 投资方, 领投方, 融资金额, 融资轮次, 披露时间, 事件时间

* **公司上市 (Company Listing)**
    * **角色**：上市公司, 证券代码, 环节 (枚举值：筹备上市, 正式上市, 暂停上市, 终止上市), 发行价格, 募资金额, 披露时间, 事件时间, 市值

* **企业收购 (Enterprise Acquisition)**
    * **角色**：收购方, 被收购方, 收购金额, 披露时间, 事件时间

* **中标 (Bidding)**
    * **角色**：中标公司, 招标方, 中标金额, 披露时间, 中标时间, 中标标的

* **高管变动 (Senior Manager Changes)**
    * **角色**：高管姓名, 任职公司, 变动类型 (枚举值：辞职, 聘任), 职位, 变动时间, 披露时间, 事件时间

* **解除质押 (Cancellation of Pledge)**
    * **角色**：质押方, 质权方, 质押物, 解除质押股份数量, 披露时间, 事件时间

* **股东减持 (Stock Reduction)**
    * **角色**：减持方, 减持金额, 减持股份数量, 减持部分占所持股份比例, 减持完成时间, 披露时间, 事件时间

* **股东增持 (Equity Overweight)**
    * **角色**：增持方, 增持金额, 增持股份数量, 增持部分占所持股份比例, 增持完成时间, 披露时间, 事件时间

* **企业破产 (Go Bankruptcy)**
    * **角色**：破产公司, 破产时间, 披露时间, 事件时间

* **企业亏损 (Enterprise Loss)**
    * **角色**：亏损公司, 亏损金额, 财报周期, 披露时间, 事件时间

* **被约谈 (Be Grilled)**
    * **角色**：被约谈方, 约谈机构, 约谈时间, 披露时间, 事件时间

## 4. 数据格式与样例

数据集文件为 JSON格式，每一行代表一个文档及其对应的标注信息。

### 文件说明
* `duee_fin_train.json`：模型训练数据。
* `duee_fin_dev.json`：模型验证数据。
* `duee_fin_sample.json`：小样本调试数据。
* `duee_fin_event_schema.json`：Schema 定义文件。

### 数据样例结构

每条数据包含以下核心字段：
* `id`：文档唯一标识符。
* `title`：文档标题。
* `text`：文档正文内容。
* `event_list`：事件列表，包含一个或多个事件对象。
    * `event_type`：事件类型。
    * `trigger`：触发词
    * `arguments`：论元列表。
        * `role`：论元角色。
        * `argument`：论元实体文本。

#### JSON 样例展示

```json
{
  "id": "sample_001",
  "title": "波奇宠物:拟募资8000万美元月底上市",
  "text": "宠物电商企业波奇宠物(股票代码为:“BQ”)日前更新招股书... 2014年波奇宠物曾宣布完成2500万美元B轮融资，投资方包括高盛... 2016年，波奇宠物宣布完成C轮融资，总金额1.02亿美元全额到账，招银国际领投，原有投资者高盛等及管理层均跟投...",
  "event_list": [
    {
      "trigger": "融资",
      "event_type": "企业融资",
      "arguments": [
        {"role": "事件时间", "argument": "2014年"},
        {"role": "被投资方", "argument": "波奇宠物"},
        {"role": "融资金额", "argument": "2500万美元"},
        {"role": "融资轮次", "argument": "B"},
        {"role": "投资方", "argument": "高盛"}
      ]
    },
    {
      "trigger": "上市",
      "event_type": "公司上市",
      "arguments": [
        {"role": "上市公司", "argument": "波奇宠物"},
        {"role": "发行价格", "argument": "10到12美元"},
        {"role": "募资金额", "argument": "7000万美元到8400万美元"},
        {"role": "披露时间", "argument": "日前"},
        {"role": "环节", "argument": "筹备上市"},
        {"role": "证券代码", "argument": "BQ"}
      ]
    },
    {
      "trigger": "融资",
```

1. **多事件**：包含两个“企业融资”事件和一个“公司上市”事件。
2. **多值论元**：第三个事件中，“投资方”角色对应了“招银国际”、“高盛”、“管理层”三个实体。
3. **角色共享**：“招银国际”既是“领投方”也是“投资方”。
