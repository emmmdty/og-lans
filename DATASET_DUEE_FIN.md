# DuEE-Fin 数据集说明

## 1. 文档定位

本文档仅说明当前仓库实际使用的 DuEE-Fin 数据快照、字段结构与使用注意事项。所有统计与路径均以本地 `data/raw/DuEE-Fin/` 为准，而不是以外部论文或旧版说明为准。

## 2. 当前仓库中的数据文件

当前数据目录如下：

- `data/raw/DuEE-Fin/duee_fin_train.json`
- `data/raw/DuEE-Fin/duee_fin_dev.json`
- `data/raw/DuEE-Fin/duee_fin_test.json`
- `data/raw/DuEE-Fin/duee_fin_sample.json`
- `data/raw/DuEE-Fin/duee_fin_event_schema.json`
- `data/raw/DuEE-Fin/duee-fin 事件类型及对应角色.pdf`

其中，JSON 文件是训练与评测直接依赖的数据；PDF 更适合作为人工核对 schema 的辅助材料，不是代码主入口。

事件类型如下：

- 解除质押
- 股份回购
- 股东减持
- 亏损
- 中标
- 高管变动
- 企业破产
- 股东增持
- 被约谈
- 企业收购
- 公司上市
- 企业融资
- 质押

本仓库中的训练、推理后处理与评测逻辑都默认围绕这 13 类事件展开。

## 6. 样本结构

数据文件按 JSONL 组织，每行一个文档对象。训练与开发集常见字段如下：

- `id`：文档唯一标识
- `title`：标题
- `text`：正文
- `event_list`：事件列表

每个事件对象通常包含：

- `event_type`：事件类型
- `trigger`：触发词
- `arguments`：论元列表

每个论元对象通常包含：

- `role`：角色名
- `argument`：论元文本

这一定义与 `src/oglans/data/adapter.py`、`src/oglans/data/prompt_builder.py`、`src/oglans/utils/json_parser.py` 中的处理逻辑保持一致。

## 7. 与当前代码的关系

当前仓库对 DuEE-Fin 的使用方式如下：

- `DuEEFinAdapter` 从 `train/dev/test` 读取 JSONL，并将 `event_list` 转换为训练或评测所需结构。
- `ChinesePromptBuilder` 围绕 DuEE-Fin 的 13 类事件生成中文提示词和结构化目标输出。
- `DS-CNS` / `OG-LANS` 通过 `duee_fin_event_schema.json` 构建事件类型图谱，用于负样本难度控制。
- `evaluate.py` 和 `evaluate_api.py` 依赖 schema 做角色合法性检查、schema compliance 统计与 CAT-lite 过滤。

## 8. 使用注意事项

- `test` split 当前是 prediction-only 数据，不应在论文中报告其 F1。
- 数据为中文金融文本，存在时间敏感性和实体敏感性；自动抽取结果不应直接用于投资、合规或法律决策。
- LLM 可能生成 schema 外事件、虚构论元或文本中不存在的值，因此仓库内额外实现了 JSON 修复、schema 检查、CAT-lite 与反事实一致性评估。
- 本仓库不重新定义 DuEE-Fin 的上游授权条款；数据使用与再分发仍应遵守原始发布方约束。
