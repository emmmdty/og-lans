# AGENTS.md

本文件用于约束 Codex CLI 在当前仓库内的长期行为。主执行计划见 `OG-LANS_ExecPlan.md`。

## 1. 事实来源
只允许依赖：
- 当前工作树
- `plan_A_首选_结构约束解码_证据对齐_v2.md`
- `og-lans-deep-research-report.md`
- `OG-LANS_ExecPlan.md`
- `RUNBOOK_TEMPLATES.md`

禁止事项：
- 禁止假设仓库结构、命令参数、配置键、函数名

若某事实无法从当前工作树或上述文档确认，统一写：
- `TODO: inspect repo`
- `待 Codex CLI 进入仓库后确认`
- `若仓库不支持则降级为……`

## 2. 固定约束
- 基座模型：`Qwen-4B-instruct-2507`
- 环境：`torch2.6.0+cu118`
- 硬件：单卡 `A6000`
- 数据集：`DuEE-Fin`
- 当前工程主线：`OG-LANS`
- F1 只允许基于 dev
- test 不得用于 F1 汇报

## 3. 工作流纪律
1. 先 inspect repo，再设计改动，再执行改动。
2. 先做 MVP，再做增强版。
3. baseline 回退路径必须保留。
4. 每完成一个阶段，都要写 run manifest / phase note。
5. 任何结果都必须标记 provenance：
   - `LOCAL_RUN`
   - `LOCAL_REEVAL`
   - `REFERENCE_ONLY`

## 4. 当前优先方向
本轮首选方向不是重 SCV，而是：
- 结构约束解码
- Schema-Aware 修复
- 证据对齐 / grounding
- SCV-Lite（只处理 grounding 失败、互斥事件、共享角色归属冲突）

## 5. 结果可信度规则
- 没有 run_id / manifest / summary 的结果，不得进主表。
- 口径不清（特别是 dev/test 不清）的结果，不得进 F1 表。
- 无法直接复现的外部 baseline，只能标 `REFERENCE_ONLY`。
- 离线 API 产物不存在时，不得触发新的在线调用。

## 6. 修改纪律
- 不得同时并行修改同一批文件。
- 改动 `evaluate.py` 前先确认旧 summary schema。
- 改动 `prompt_builder.py` 时必须保留旧 prompt 路径或回退点。
- 改动 `json_parser.py` 时，新增字段尽量放 sidecar diagnostics，避免破坏既有评测输入。
- 改动 `scv.py` 时必须同时保留 `SCV-on`、`SCV-off`、`SCV-Lite` 三种可区分状态。

## 7. 每次 run 必存信息
- 配置快照
- 启动命令
- 代码改动摘要
- 训练/推理耗时
- 显存占用（若可获得）
- dev 集指标 summary
- 失败样本统计
- 错误类型统计
- API usage / retries / manifest（若适用）

## 8. 停止条件
遇到以下情况立即停止并记录 blocker：
- 无法确认 dev 集或评测口径
- 无法确认 baseline 或 evaluate 入口
- 必须联网才能继续
- 关键改动破坏 baseline 回退路径
- 结果无法追溯到本地文件或来源说明

## 9. 并行原则
允许 subagents 并行做只读核查：
- 代码入口核查
- 结果目录清点
- 评测脚本与指标口径核对

禁止：
- 并行修改同一文件
- 在事实未合并前分别实现不同方案

## 10. 输出要求
最终必须能交付：
- 实验执行 summary
- 主实验结果表
- 消融结果表
- 效率/成本结果表
- 可靠性结果表
- 失败与风险说明
- 论文主表建议
- 附录消融建议
- 后续 backlog
