# OG-LANS 项目说明（论文叙事版）

## 1. 研究问题

本项目研究的核心问题是：在金融篇章级事件抽取任务中，如何让大语言模型在严格结构化输出约束下，同时保持较高的论元级抽取质量、较低的幻觉率和较强的 Schema 合规性。

与开放式生成不同，金融事件抽取存在三个更尖锐的约束：

1. 输出必须映射到固定事件类型和角色集合。
2. 错误往往不是“整句不通顺”，而是细粒度角色错配、边界偏移或数值幻觉。
3. 模型既要读长文档，又要在 JSON 级别给出可评估结果。

因此，本项目不把问题建模为单纯的指令微调，而是把它建模为一个“结构化偏好学习 + 动态难度控制 + 一致性诊断”的组合问题。

## 2. 方法论动机

标准 SFT 在这类任务上常见两个问题：

1. 模型容易学会输出看似规范的结构，但事实内容不稳定。
2. 模型对“轻微错误”和“严重幻觉”缺乏区分能力。

标准 DPO 虽然提供了正负偏好对比机制，但它的上限受限于负样本质量。如果负样本过于容易，模型会退化为识别模板差异；如果负样本质量不受控，偏好学习就会学到错误边界。

OG-LANS 的方法论出发点是：不把负样本当作静态资源，而把它当作一个需要随着模型能力动态变化的训练信号生成过程。

## 3. 方法总体框架

当前项目将方法分解为四个层面：

1. 偏好训练主干
2. 本体图驱动的负样本构造
3. 能力感知的难度调度
4. 推理与结果一致性诊断

四层之间的关系如下：

- 偏好训练负责让模型偏向正确结构化输出。
- 负样本构造负责定义“什么是困难但合理的错误”。
- 动态调度负责决定在什么训练阶段给模型什么难度的负样本。
- 一致性诊断负责检验模型是否只是形式上正确，还是在推理和结果层面都更可信。

## 4. 核心算法思想

### 4.1 偏好优化主干

项目当前以 IPO/DPO 风格训练为主线，在工程上由 `CGADPOTrainer` 承载。其思想不是直接最小化 token 级监督误差，而是比较“优选输出”和“劣选输出”的相对偏好。

这种建模适合事件抽取任务的原因在于：

1. 同一输入下可以显式构造正确输出与错误输出。
2. 错误输出不只有一种，因此可以形成更接近真实决策边界的对比学习信号。
3. 当训练目标转为“正确输出相对更优”时，模型更容易学习结构化判别边界，而不是只记住单一模板。

### 4.2 本体图驱动负样本

项目不满足于做简单随机扰动，而是用本体图距离和事件结构关系控制负样本强度。其目的在于把负样本组织成不同难度层次：

1. easy negative：与目标事件关系较远、较容易排除的错误
2. medium negative：局部接近、但结构不匹配的错误
3. hard negative：在事件类型、角色或数值层面与正确结果高度相似的错误

这一设计的理论意义在于：模型真正需要学习的不是“什么明显是错的”，而是“什么看起来像对的但实际上不成立”。

### 4.3 OG-LANS 动态调度

OG-LANS 的核心创新不是单纯引入 hard negative，而是根据训练过程中的损失变化估计当前能力值，并据此动态切换负样本难度分布。

直观上：

1. 训练初期，模型尚未掌握基本模式，应避免过多 extremely hard negative。
2. 随着能力上升，继续喂 easy negative 的边际收益会迅速下降。
3. 当模型已经能轻松区分简单错误时，训练信号应逐步迁移到更细粒度、更具迷惑性的负样本上。

这使得课程学习不再依赖固定步数表，而是依赖模型当前的可观测状态。

### 4.4 SCV 语义一致性验证

在偏好学习中，一个常被忽略的问题是“伪负样本”。如果某个被构造成负样本的结果在语义上仍可能成立，那么它就会污染偏好边界。

SCV 的作用就是用语义一致性验证去过滤这类风险样本。其学术意义在于：

1. 它降低了偏好数据噪声。
2. 它使 hard negative 更接近真正的“反例”而非随机构造错误。
3. 它为后续将训练失败归因于模型能力，而非归因于数据标签污染，提供了更强的解释基础。

### 4.5 CoT 与结果一致性诊断

当前项目并未把 CoT 当作最终目标，而是把它当作一个诊断视角。模型可能输出正确 JSON，但其中间解释未必忠实；反过来，它也可能说得很像样，但结果仍然错。

因此当前评估除了主任务 F1 外，还显式记录：

1. CoT 忠实度
2. CoT 类型一致性
3. CoT 论元一致性
4. 反事实一致性

这使项目不再只问“结果对不对”，而是进一步问“模型是如何到达这个结果的”。

## 5. 当前工程实现如何支撑方法论

项目的方法论并不是停留在概念层，而是已经落到明确的代码分层中。

方法与代码的映射关系如下：

1. 偏好训练器
   - `src/oglans/trainer/unsloth_trainer.py`
2. 能力调度与负样本策略
   - `src/oglans/utils/ds_cns.py`
3. 语义一致性校验
   - `src/oglans/utils/scv.py`
4. 提示词与正负样本模板
   - `src/oglans/data/prompt_builder.py`
5. 本地评估
   - `evaluate.py`
6. API 评估
   - `evaluate_api.py`
7. 反事实扰动与 CAT-lite 流水线
   - `src/oglans/inference/cat_lite.py`

这种分层的好处是：每一个研究假设都有对应的实现位置和实验开关，便于做消融、回滚和误差归因。

## 6. 当前实验设计逻辑

当前项目的实验设计遵循“本地模型、基座模型、API 基线”三方对照。

### 6.1 本地微调模型

这是项目的主要研究对象，用于检验 OG-LANS 训练机制是否真的带来增益。

### 6.2 纯基座模型

这是必要对照。加入 `base-only` 评估之后，当前项目可以在同一套本地评估协议下比较：

- 未微调的 Qwen 基座
- 微调后的 LoRA 模型

这使得“训练是否真的改进了结构化抽取能力”可以被直接回答。

### 6.3 API 基线

API 基线的作用不是证明本地方法一定更强，而是提供一个外部高性能参照。项目当前允许在统一协议下评估 zero-shot、few-shot 和多 seed 设置，以减少仅用单次 API 结果得出结论的偶然性。

## 7. 当前项目的主要创新点

若按论文叙事组织，当前项目可主张的创新点主要有三类。

### 7.1 训练信号层创新

负样本不是固定静态集合，而是由本体图结构、难度分层与能力调度共同决定。这把事件抽取中的偏好学习，从“比较两个答案”扩展成“比较两种带难度控制的结构化答案”。

### 7.2 课程学习层创新

项目不是预设一个固定难度时间表，而是通过损失驱动的能力估计实现在线课程调整。这使课程学习和模型真实训练状态绑定，而不是仅与 step 编号绑定。

### 7.3 诊断层创新

项目没有把学术评估停留在 strict/relaxed/type F1，而是同时记录 hallucination、schema compliance、CoT 一致性和反事实指标。对于事件抽取这类高约束任务，这种多维诊断比单一 F1 更能支持后续方法分析。

## 8. 当前项目的实验价值

从学术角度看，这个项目的价值不只在于“最后分数是否超过某个 API”，还在于它建立了一套可研究的实验平台：

1. 可以研究 hard negative 的真正收益来自哪里。
2. 可以研究结构化抽取中 CoT 与最终结果之间是否一致。
3. 可以研究在同一协议下，本地微调模型、纯基座模型和商业 API 的差异来源。
4. 可以研究 schema compliance 与 strict F1 之间的张力。

这使项目不仅能产出结果，也能产出方法学讨论。

## 9. 当前项目的局限

当前版本仍存在明确局限，这些局限也是后续论文必须诚实交代的部分：

1. 任务范围集中在金融事件抽取，跨数据集泛化尚未系统验证。
2. SCV 与长文本推理带来较高运行成本。
3. CoT 相关指标目前更接近内部一致性诊断，而不等同于严格因果忠实度测量。
4. 即便训练框架已成型，最终性能仍需通过严格协议和多 seed 结果来证明。
